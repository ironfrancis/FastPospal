# 安全政策

## 支持的版本

| 版本 | 支持 |
|------|------|
| `main` 分支最新代码 | ✅ |
| 已发布 Release / PyPI 版本 | ✅（发布后维护） |
| 其他分支或 fork | ❌ |

## 报告漏洞

如果你发现 FastPospal 中的**安全漏洞**（例如凭证泄露、鉴权绕过、会话固定、敏感信息写入日志等），请**不要**在公开 Issue 中讨论。

请通过以下方式私下报告：

1. **GitHub Security Advisories**（推荐）  
   在仓库页面选择 **Security → Advisories → Report a vulnerability**

2. **私信维护者**  
   通过 GitHub 向仓库维护者发送私信，标题请注明 `[FastPospal Security]`

报告时请尽量包含：

- 漏洞类型与影响范围
- 复现步骤或 PoC（请使用测试账号，勿使用真实生产数据）
- 受影响版本或 commit
- 可能的修复建议（如有）

我们会在 **7 个工作日内**确认收到，并在可行范围内协调修复与披露时间。

## 不在本政策范围内

以下情况请改用普通 Issue，而非安全报告：

- 银豹官方 Web 后台或开放平台自身的漏洞
- 功能缺陷、解析错误、接口变更导致的数据不一致
- 未授权访问他人门店的需求或咨询
- 绕过银豹认证、破解、批量撞库等恶意用途

FastPospal 仅面向**合法授权账号**的自动化集成，不提供、也不接受任何破坏第三方系统安全的贡献。

## 安全使用建议

- 勿将 `.env`、`.pospal_session.json` 或 `MCP_AUTH_TOKEN` 提交到版本库
- 远程 HTTP 部署务必配置 `MCP_AUTH_TOKEN` 与 `FASTMCP_HTTP_ALLOWED_HOSTS`
- 写操作（创建/更新/删除商品、会员等）请在测试门店或明确授权环境中进行
- 生产环境定期轮换 Token 与登录密码
